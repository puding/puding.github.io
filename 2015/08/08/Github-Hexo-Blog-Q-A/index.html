Github + Hexo 疑難雜症

前言

這是利用 Github + Hexo 建立的 Blog，
以下內容是一位新手在建置此站過程中，受到打擊後的Mermer

本篇會專門講述，所遇到的問題與解決方法，
參考的格式如下：

有任何更好的方式或錯誤，歡迎分享告知:D

1. 如何建立留言板

處理狀態：已解決(發現新問題)

依目前自己搜尋到的結果主要大宗分兩種『多說』與『Disqus』，
本站選擇了『Disqus』

解決方法：
做法：只要在Hexo設定檔_config.yml設定以下參數即可

但可能因為各家寫教學文時間點的關係，
在『Disqus』官網，我一直找不到shortname的存在阿阿阿阿阿!!!!((( 抱頭~~ My English is very poor )))
於是胡亂設定的結果，留言板就出現以下訊息： ( 要上傳到Github才看的到喔，本機看不到 )

解決方法(續)：
在Disqus申請一組帳號後，直接進入申請網站 ，申請一個Site name，它就是disqus_shortname要設定的值
請注意Site name設定之後，不能更改

說明網址：What’s a shortname?

發現新問題：更換主題，無法使用『Disqus』

2. 快速發佈

處理狀態：已解決

做任何動作(新增、修改、刪除 文章及設定)之後，都要再執行兩個指令，才能在本機預覽畫面。

步驟 指令 簡碼 作用
1 hexo generate hexo g 產生靜態檔案至public
2 hexo server hexo s 啟動伺服器。使127.0.0.1:4000可以預覽Blog畫面

解決方法：
因為這兩個步驟，會經常性使用，所以我會開兩個 git bash ( 或 git shell ) 視窗，常駐這兩個指令，如下：

hexo g -w，這個指令，能監控檔案的任何異動，自動產生檔案至public，詳細可參考：Hexo 指令說明

PS: 如果有更好的方式，再跟分享給我XD
